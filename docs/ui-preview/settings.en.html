Settings

A dedicated settings page for project-level personal configuration. Save applies immediately and returns reloaded modules and next-run-only keys.

Standalone Page Current route: /docs/ui-preview/settings.en.html

Configuration Description

  • Storage: write paths for tool runtime data. Updates affect future writes only; no automatic migration.
  • Workspace: target project path and git-root strategy for run/code-task execution context.
  • Test Assets: shared/generated test roots and inclusion switches for default runs.
  • Diagnostics: correlation-key extraction rules and time window, affecting trace/log hit rate.
  • Trace / Logs: provider and endpoint. Query-related capabilities apply immediately after save.
  • AI / CodeAgent: model and remediation policy. Some keys may be next-run only.
  • Report / UI: report port and locale; port change applies on next app start.
