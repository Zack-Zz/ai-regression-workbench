# zarb Operator Guide

## Installation

This repository is a monorepo. The CLI is not yet published to npm. Install locally:

```bash
git clone https://github.com/your-org/ai-regression-workbench.git
cd ai-regression-workbench
pnpm install
pnpm build
# Link the CLI globally so `zarb` is available in PATH
cd apps/cli && npm link
```

Or run directly without linking:

```bash
node apps/cli/dist/bin.js
```

## First-run setup

```bash
zarb init    # creates .zarb/ with default config
zarb doctor  # verify environment dependencies
zarb         # start server; open http://127.0.0.1:3000 in your browser
```

The server serves both the API and the bundled local workbench UI from the same port. No separate UI process is required.

## Configuration

Edit `.zarb/config.local.yaml` (generated by `zarb init`):

```yaml
storage:
  sqlitePath: .zarb/data/sqlite/zarb.db

report:
  port: 3000

workspace:
  targetProjectPath: /path/to/your/project   # required for code tasks

ai:
  apiKeyEnvVar: OPENAI_API_KEY               # env var name, not the key itself

trace:
  provider: jaeger                           # or: none
  endpoint: http://localhost:16686

logs:
  provider: loki                             # or: none
  endpoint: http://localhost:3100

# Logging configuration
log:
  level: info    # debug | info | warn | error
  file: true     # write logs to .zarb/logs/zarb.log
```

## Logging

zarb has a built-in structured logger (`AppLogger`) that outputs to console and optionally to a file.

### Log levels

| Level | Content |
|-------|---------|
| `error` | Fatal errors: playwright startup failure, AI request exception, task execution crash |
| `warn` | Non-fatal: click/fill failure, LLM empty response, adapter errors |
| `info` | Business key points: run created/completed/failed, AI call duration, task status, session lifecycle |
| `debug` | Details: every playwright operation (navigate/click/fill), every tool call record |

### Configure via config file

```yaml
log:
  level: debug   # show playwright operation details
  file: true
```

Restart `zarb` after changing the config file.

### Configure via environment variable

The environment variable takes precedence over the config file and takes effect immediately without restart:

```bash
ZARB_LOG_LEVEL=debug zarb
```

### Log file location

`.zarb/logs/zarb.log` — NDJSON format, one record per line.

```bash
# Tail the log file
tail -f .zarb/logs/zarb.log | jq .

# Filter by level
tail -f .zarb/logs/zarb.log | jq 'select(.level=="error")'

# Filter by module
tail -f .zarb/logs/zarb.log | jq 'select(.module=="PlaywrightTool")'
```

## Security guidelines

- **Never store API keys in config files** — use `apiKeyEnvVar` to reference an environment variable name
- **Workspace path** — `targetProjectPath` must be an absolute path to a git repository; relative paths and path traversal (`../`) are rejected at commit time
- **Commit scope** — commits only stage files listed in `changedFiles` (system-derived); unrelated dirty files are never included
- **Review gate** — `COMMIT_PENDING` requires explicit human review acceptance; verify-failed tasks require `forceReviewOnVerifyFailure=true`
- **Git credentials** — zarb uses the system git credential store; no credentials are stored by zarb itself

## Dependency requirements

| Dependency | Required | Notes |
|---|---|---|
| Node.js | ≥ 22 | |
| git | required | for diff/patch/commit |
| Playwright | required for test runs | `npx playwright install` |

| kiro-cli | optional | for interactive repair sessions |

## Database migrations

Migrations run automatically on `zarb init` and on every `zarb` startup. To check schema health:

```bash
zarb doctor
```

## Uninstall

```bash
npm uninstall -g ai-regression-workbench
rm -rf .zarb   # remove local data (optional)
```
